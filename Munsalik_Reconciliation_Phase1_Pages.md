# Munsalik 对账后台一期页面方案

**版本：** V1.2（轻量人工处理版）  
**日期：** 2026-07-20  
**范围：** 仅规划运营后台页面、字段、按钮和交互，不包含邮件接收、文件解析、自动匹配及数据修正的后台实现。

---

## 1. 一期设计原则

业务尚未正式运行，暂时无法判断实际差异量。第一期页面以“查看对账结果、人工核实、记录结论”为目标，不在对账页面执行销账、冲正或直接修改业务数据。

确认以下产品原则：

1. Munsalik 文件发送频率不固定，不按照固定 T+1 展示或考核。
2. 不做对账工作台或数据概览页。
3. 不做后台文件上传入口；补充文件及修正版通过白名单邮箱重新发送。
4. 不做独立的差异审批、销账、冲正及账务执行页面。
5. 不做规则与模板配置页面。
6. 对账页面不允许直接修改放款、还款或账务数据。
7. 我方数据需要修正时，转到对账页面之外通过现有业务功能或修正工单处理；对账页面只记录修正进度和结果。
8. “关闭差异”只代表人工核实流程结束，不代表双方数据已经一致，也不代表账务数据已经修正。
9. 新文件补齐原异常后，待处理异常自动消失；历史差异保留并标记为“系统自动解决”，不做物理删除。

## 2. 一期页面结构

```text
财务与对账
├── 对账任务
│   ├── 任务详情
│   ├── 对账明细
│   └── 差异人工处理
├── 邮件隔离区
└── 审计日志
```

一期不单独设置“对账明细”“差异处理”和“异常告警”菜单。

- 对账明细和差异处理放在任务详情中，通过页签查看。
- 文件校验失败、处理失败等异常直接在任务列表中用状态提示。
- 被隔离的邮件统一进入“邮件隔离区”。
- 如果后续任务量或差异量明显增大，再拆分独立菜单。

用户进入“财务与对账”后默认打开“对账任务”。

## 3. 对账任务列表

### 3.1 页面用途

查看每份放款或还款对账文件的基本信息、处理状态及结果汇总。

### 3.2 查询条件

| 查询项 | 页面组件 |
|---|---|
| 任务编号 | 输入框 |
| 文件类型 | 下拉框：放款、还款 |
| 文件覆盖日期 | 日期范围 |
| 来源 | 下拉框：Munsalik 官方邮件、我方白名单邮箱补发 |
| 发件人 | 输入框 |
| 收件时间 | 日期时间范围 |
| Message-ID | 输入框 |
| 文件名 | 输入框 |
| SHA-256 | 输入框 |
| 任务状态 | 多选下拉框 |

页面按钮：

- `查询`
- `重置`
- `导出当前结果`（有权限时显示）

### 3.3 列表字段

| 字段 | 展示说明 |
|---|---|
| 任务编号 | 点击进入任务详情 |
| 文件类型 | 放款/还款 |
| 文件覆盖日期 | 开始日期至结束日期 |
| 来源 | Munsalik 官方邮件/我方白名单邮箱补发 |
| 发件人 | 实际发件邮箱 |
| 收件时间 | 邮件到达时间 |
| 文件名 | 超长省略，悬浮展示完整名称 |
| SHA-256 | 默认截断，支持复制 |
| 任务状态 | 使用颜色标签展示 |
| 总笔数 | 文件有效交易笔数 |
| 匹配笔数 | 完全一致笔数 |
| 差异笔数 | 非完全一致笔数 |
| 待人工处理 | 尚未关闭的可处理差异数量 |
| 操作 | 查看详情 |

### 3.4 任务状态

- 处理中
- 已完成
- 校验失败
- 处理失败
- 已隔离
- 重复文件已忽略

展示规则：

- 已完成：绿色
- 处理中：蓝色
- 校验失败、处理失败：红色
- 已隔离：橙色
- 重复文件已忽略：灰色

任务列表不展示“T+1 超期”或“日报漏发”状态。

## 5. 差异人工处理

### 5.1 处理目标

差异处理只完成以下事项：

1. 记录运营与 Munsalik 的核实过程。
2. 记录对方最终确认结果。
3. 记录差异属于对方问题、我方问题或口径问题。
4. 等待并关联后续补充对账文件。
5. 我方数据需要修正时，登记外部修正工单及进度。
6. 核实完成后关闭差异。

差异页面不执行销账、冲正、退款、补账或数据库修改。

### 5.2 处理状态

| 状态 | 含义 |
|---|---|
| `PENDING` | 待人工处理 |
| `PROCESSING` | 人工核实中 |
| `WAITING_PARTNER` | 等待 Munsalik 回复 |
| `WAITING_SUPPLEMENT` | 等待补充对账文件 |
| `PENDING_INTERNAL_FIX` | 等待我方数据修正 |
| `INTERNAL_FIXED` | 我方已完成修正，等待复核结果 |
| `AUTO_RESOLVED` | 新文件补齐后由系统自动解决 |
| `CLOSED` | 人工核实流程已结束 |

### 5.3 页面操作按钮

根据状态显示：

- `开始处理`
- `登记处理结果`
- `标记等待对方回复`
- `标记等待补充文件`
- `转我方数据修正`
- `登记修正结果`
- `关闭差异`

不显示：

- 销账
- 冲正
- 通用调账
- 提交财务复核
- 执行账务
- 直接修改数据

### 5.4 不同差异方向的人工处理

| 场景 | 页面处理方式 |
|---|---|
| 放款：我方有、对方无 | 联系对方核实；确认有则等待补充文件；确认无则上传凭证，记录确认结果后关闭 |
| 放款：我方无、对方有 | 联系对方核实；对方报表错误则记录凭证后关闭；对方记录有效则转我方数据修正或内部核查 |
| 还款：我方有、对方无 | 联系对方核实；确认有则等待补充文件；确认无则上传凭证，记录确认结果后关闭 |
| 还款：我方无、对方有 | 联系对方核实；对方确认无则上传凭证后关闭；对方记录有效则转我方数据修正或内部核查 |
| 金额不一致 | 确认双方正确金额；我方错误则转我方数据修正，对方错误则等待补充文件或确认后关闭 |
| 状态不一致 | 确认双方最终状态；我方错误则转我方数据修正，对方错误则等待补充文件或确认后关闭 |
| 重复记录 | 确认重复来源及有效记录，登记处理结论后关闭；页面不直接删除数据 |

### 5.5 差异详情抽屉

#### A. 差异概要

- 差异编号
- 业务类型
- 差异方向
- 原始对账状态
- 当前处理状态
- 所属任务、文件和 Excel 行号
- 创建时间、更新时间

#### B. 双方数据对比

复用对账明细的左右对比组件，不一致字段突出显示。

#### C. 人工处理信息

- 对方确认结果
- 问题归属
- 处理结论
- 沟通时间
- 沟通方式
- 对方回复内容
- 凭证附件
- 处理备注
- 经办人

#### D. 我方数据修正信息

仅转我方修正时展示：

- 需要修正的业务对象
- 需要修正的字段
- 当前值
- 期望值
- 修正原因
- 修正工单号
- 修正负责人
- 修正状态
- 实际修正前值、修正后值
- 实际执行人
- 完成时间
- 工单或脚本执行凭证

这些字段只记录修正信息，不在本页面直接修改业务数据。

#### E. 自动解决信息

新文件重新匹配并补齐原异常后，只读展示：

- 解决方式：新文件自动补齐
- 自动解决时间
- 最新对账任务编号
- 最新文件名及收件时间
- 最新匹配结果

状态变为“系统自动解决”后，该记录默认不再出现在“待处理”列表中，但可以通过“全部”或“系统自动解决”状态查询。

#### F. 处理时间线

按时间顺序展示开始处理、联系对方、上传凭证、等待补充文件、转修正工单、登记修正结果和关闭等操作。

### 5.6 登记处理结果弹窗

字段：

| 字段 | 要求 |
|---|---|
| 对方确认结果 | 必填：确认有、确认无、对方报表错误、尚未确认 |
| 问题归属 | 必填：对方问题、我方问题、双方口径问题、暂无法判断 |
| 处理结论 | 必填，见下方枚举 |
| 沟通时间 | 必填 |
| 沟通方式 | 必填：邮件、电话、即时通讯、其他 |
| 对方回复内容 | 必填 |
| 凭证附件 | 关闭高风险差异时必填 |
| 处理备注 | 选填 |

处理结论：

- 等待对方回复
- 等待补充文件
- 转我方数据修正
- 对方报表问题，不修正我方数据
- 双方已确认差异，暂不修正
- 我方数据已修正
- 无需继续处理

### 5.7 转我方数据修正弹窗

当确认我方数据需要修正时使用。

字段：

- 业务对象：放款记录、还款订单、账务记录、其他
- 业务单号
- 需要修正的字段
- 当前值
- 期望值
- 修正原因
- 修正工单号（创建工单后必填）
- 修正负责人
- 相关凭证
- 备注

提交后状态变为“等待我方数据修正”。

### 5.8 登记修正结果弹窗

字段：

- 修正工单号
- 修正结果：成功、失败、无需修正
- 实际修正前值
- 实际修正后值
- 实际执行人
- 完成时间
- 工单、脚本或审批记录附件
- 结果说明

登记成功后状态变为“我方已完成修正”，由运营重新查看双方数据并决定是否关闭。

### 5.9 关闭差异弹窗

字段：

- 关闭结论（必填）
- 关闭说明（必填）
- 对方确认结果
- 问题归属
- 凭证附件
- 修正工单号及修正结果（存在我方修正时展示）

关闭结论：

- 对方报表问题，已确认
- 我方数据已修正
- 双方确认差异，暂不修正
- 无需处理
- 其他

弹窗需明确提示：

> 关闭差异仅表示人工核实流程结束，不会修改放款、还款或账务数据。

## 11. 一期页面验收重点

1. 菜单只保留对账任务、邮件隔离区和审计日志。
2. 不出现数据概览、固定 T+1、后台上传、销账、冲正及审批页面。
3. 对账任务能够查看结果汇总、明细、差异、原邮件和附件信息。
4. 对账明细能够左右对比双方字段，不允许在线编辑。
5. 差异可以登记对方确认结果、问题归属、处理结论及凭证。
6. 差异可以标记等待对方回复或等待补充文件。
7. 我方数据需要修正时，可以记录修正字段、工单号、负责人和修正结果，但不能在对账页面直接改数据。
8. 新文件补齐原异常后，异常自动从待处理列表消失，并在历史记录中标记为“系统自动解决”。
9. 系统自动解决的历史差异仍可查询，不允许直接删除。
10. 关闭差异时必须填写关闭结论，并明确提示关闭不会修改业务数据。
11. 所有处理、附件查看、导出、自动解决和关闭动作可在审计日志中查询。
