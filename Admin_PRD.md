# Anole Loan 巴基斯坦借贷业务后台 PRD

**版本：** V1.2  
**日期：** 2026-06-05  
**产品：** Anole Loan 内部运营后台  
**适用范围：** 管理员权限、客户授信、借款订单、账单费用、放款还款、逾期、催收管理、优惠券、财务对账、业务配置、App资源位管理

---

## 1. 背景与目标

Anole Loan 面向巴基斯坦用户提供移动端现金借贷服务。随着业务量增长，内部需要一套可控、可审计、可按角色分权的运营后台，支撑风控审核、客服查询、催收处理、财务对账、运营配置和系统管理。

本后台的核心目标：

- 建立标准 RBAC 权限体系，控制不同岗位可见菜单和可执行操作。
- 围绕用户、授信、借款、账单、还款、逾期、优惠券、配置等业务表，提供查询、处理和审计能力。
- 为研发对接接口、后续迭代和权限拆分提供统一产品口径。

## 5. 功能需求

### 5.1 仪表盘

- 展示关键指标：总客户数、在贷余额、逾期订单数、待审核授信、今日放款、今日还款、待处理费用调整、待处理对账。
- 所有金额使用 PKR。
- 指标需支持按自然日统计，时区采用 PKT（UTC+5）。

### 5.2 客户管理

**客户列表**

- 字段：客户姓名、手机号、证件号、授信状态、总额度、已用额度、钱包类型、黑名单状态、注册时间。
- 筛选：姓名/手机号/证件号、授信状态、黑名单状态。
- 操作：查看详情、加入/移出黑名单、导出。
- 证件号、手机号默认脱敏；具备敏感信息权限时可查看完整值。

**客户详情**

- 基础信息：姓名、手机号、证件号、城市、注册时间、钱包信息、黑名单状态。
- 授信信息：授信状态、审核状态、额度、已用额度、锁定额度、审核类型、审核时间、拒绝原因。
- 身份认证：证件正反面、生日、性别、省市、详细地址。
- 联系人：关系、姓名、手机号。
- 授信记录：同一客户可能存在多次授信申请，客户详情中需独立展示 `usr_credit` 申请历史，字段包括申请单号、提交时间、授信状态、当前步骤（基础信息/身份认证/紧急联系人/等待审核/已完成）、审核类型、审核结果、审核时间、授信额度、已用额度、过期时间、下次可申请时间、拒绝原因、备注；申请中的记录必须能展示客户当前停留在哪一步。
- 授信变更：展示 `usr_credit_change_log`，字段包括变更类型、变更金额、变更前后额度、关联订单、备注、时间。

### 5.3 授信审核

- 展示待审核、已通过、已拒绝、人工复核等状态。
- 审核员可查看客户基础信息、身份信息、联系人、历史借款和授信变更。
- 操作：通过、拒绝、转人工复核。
- 拒绝必须填写原因。
- 所有审核操作写入后台审计日志和授信变更记录。

### 5.4 借款订单

- 字段：借款订单号、客户、客户借款次数（首次/第 2 次/第 3 次/第 4 次等）、借款金额、合同本金、分期期数、每期天数、日利率、逾期费率、总利息、总服务费、展示应还金额、订单状态、渠道、放款时间、首期还款日、最后还款日。
- 筛选：订单号/手机号、订单状态、渠道、申请时间。
- 操作：查看详情、审批/拒绝、人工放款；当订单状态为 `CoolingOff` 时，在借款详情中支持“打开提前结清入口”操作，入口默认关闭，操作后客户侧/还款侧可进入提前结清流程，并在后台提前结清记录中可按借款订单号定位。
- 订单状态建议统一为：`PendingApproval`、`PendingDisbursement`、`CoolingOff`、`Active`、`Overdue`、`Paid`、`WrittenOff`、`Rejected`。

### 5.5 放款记录

- 字段：放款单号、借款订单号、客户、放款金额、渠道、渠道流水号、状态、请求时间、成功时间、失败原因。
- 筛选：放款单号/订单号、状态、渠道、时间范围。
- 操作：查看详情、失败重试。
- 放款重试必须记录操作人、原因和结果。

### 5.6 借款账单

- 字段：账单号、借款订单号、客户、期次、应还本金、应还利息、服务费、逾期费、已还金额、剩余金额、账单状态、到期日。
- 分期账单页仅展示账单主表 `loan_bill` 的聚合信息，不在账单详情中承载完整费用流水。
- 账单状态建议统一为：`CoolingOff`、`Active`、`PartiallyPaid`、`Overdue`、`OverduePartiallyPaid`、`Paid`、`EarlySettled`、`WrittenOff`。

### 5.7 费用明细

- 独立菜单展示 `loan_bill_fee`，用于跨客户、跨借款订单、跨账单查询费用项。
- 字段：费用号、账单号、借款订单号、客户 ID、期次、费用类型、费用名称、原始金额、应付金额、已付金额、剩余金额、减免金额、状态、生成类型、备注。
- 筛选：费用号/账单号/借款订单号、费用类型、费用状态。
- 费用类型包括：本金、利息、服务费、逾期费、优惠券抵扣。

### 5.8 费用变更

- 独立菜单展示 `loan_bill_fee_change_record`，用于查看每个账户、账单、费用项的全量变更过程。
- 字段：变更号、费用号、账单号、借款订单号、客户 ID、费用类型、变更类型、来源类型、来源单号、变更前应付、变更金额、变更后应付、实付变更、减免金额、变更原因、操作人、创建时间。
- 筛选：变更号/费用号/来源单号、变更类型、来源类型。
- 来源类型包括：逾期生成、人工调整、优惠券、还款分配。

### 5.9 放款记录

- 独立菜单展示 `disbursement_record`。
- 字段：放款单号、借款订单号、客户、放款金额、渠道、收款账户、渠道流水号、状态、成功时间、回调时间、失败原因、请求时间。
- 筛选：放款单号/借款订单号、状态、渠道。
- 失败记录需保留失败原因，支持后续增加重试操作。

### 5.10 还款订单

- 字段：还款单号、借款订单号、客户、还款金额、还款类型、渠道、钱包账号、渠道流水号、状态、对账状态、支付时间。
- 筛选：还款单号/借款订单号、状态、渠道、时间范围。
- 还款订单页展示 `repayment_order` 主表。
- 还款状态建议统一为：`Pending`、`Success`、`Failed`、`Cancelled`。

### 5.11 还款明细

- 独立菜单展示 `repayment_order_bill`。
- 字段：还款单号、借款订单号、账单号、客户 ID、期次、原账单金额、账单剩余快照、优惠券金额、应付金额、分配金额、差额、状态、支付时间、创建时间。
- 筛选：还款单号/账单号/借款订单号。

### 5.12 还款分配

- 独立菜单展示还款分配明细表。
- 字段：分配单号、还款单号、借款订单号、账单号、费用号、客户 ID、期次、费用类型、分配金额、分配顺序、创建时间。
- 筛选：分配单号/还款单号/费用号、费用类型。
- 该页面用于确认还款金额最终落到本金、利息、服务费、逾期费等哪个费用项。

### 5.13 逾期管理

- 展示逾期订单总数、1-7 天、8-30 天、30 天以上分层。
- 字段：订单号、客户、逾期本金、逾期天数、逾期费、剩余应还、到期日、渠道、催收员。
- 筛选：订单号/手机号、逾期天数区间、催收员、渠道。
- 操作：查看详情、发送提醒、记录催收备注。
- 增加每日逾期记录视图，独立展示 `loan_overdue_record`：逾期记录号、借款订单号、账单号、逾期日期、逾期天数、逾期费率、逾期本金、计费基数、费用金额、原始费用、入账费用、入账时间。

### 5.14 提前结清

- 字段：提前结清单号、借款订单号、客户、原剩余金额、减免金额、应付金额、状态、申请时间、支付时间。
- 操作：查看详情、确认结清、取消。
- 结清后需同步更新账单、费用明细、还款分配和账务流水。

### 5.15 费用调整

- 字段：调整单号、借款订单号、账单号、费用类型、调整类型、调整金额、原因、申请人、审批人、状态、创建时间。
- 支持创建费用调整，费用类型包括利息、服务费、逾期费。
- 支持审批通过/拒绝。
- 审批通过后生成费用变更记录，并同步账单剩余金额。

### 5.16 账务流水

- 字段：流水号、用户、借款订单号、账单号、业务类型、业务单号、方向、金额、币种、账户类型、账户 ID、状态、交易时间、入账时间。
- 筛选：流水号/订单号/手机号、业务类型、方向、状态、时间范围。
- 支持导出。

### 5.17 还款对账

- 字段：对账单号、还款单号、借款订单号、渠道、渠道流水号、系统金额、渠道金额、差额、对账状态、处理状态、处理人、备注。
- 对账状态：`Matched`、`Unmatched`、`AmountMismatch`、`ManualConfirmed`。
- 处理状态：`Pending`、`Handled`、`Ignored`。
- 操作：确认处理、忽略差异。
- 处理必须填写备注。

### 5.18 优惠券管理

**借款优惠券**

- 字段：券号、客户、券金额、最低借款金额、最高借款金额、适用产品、状态、生效时间、失效时间、锁定订单、使用时间。
- 操作：发券、作废。

**还款优惠券**

- 字段：券号、客户、券金额、最低剩余本金、抵扣类型、适用还款类型、状态、生效时间、失效时间、使用时间。
- 操作：发券、作废。

**使用记录**

- 展示优惠券锁定、使用、退回、过期全过程。

### 5.19 业务配置

**借款金额期数规则**

- 字段：最低借款金额、最高借款金额、可用期数、默认期数、每期天数、生效时间、失效时间、状态。
- 规则需校验金额区间不得重叠。
- 仅启用状态的规则参与用户借款匹配。

**用户期数配置**

- 可针对单个用户覆盖全局期数规则。
- 字段：用户、可用期数、期数单位、生效时间、失效时间、备注。
- 每次创建/修改必须写入变更记录。

### 5.20 催收管理

**角色权限：** COLLECTION、SUPER_ADMIN

**模块概览指标**

- 待催订单数（逾期未结清）
- 今日已跟进数
- 承诺还款待验证数
- 逾期 30+ 天严重件数

#### Tab 1：催收任务

| 字段 | 说明 |
|---|---|
| 借款订单号 | 逾期贷款单号 |
| 客户 | 姓名 + 手机号（脱敏） |
| 逾期天数 | 超出还款日的天数，30+ 标红 |
| 逾期金额 | 当前剩余应还金额（PKR） |
| 到期日 | 账单原始到期日 |
| 承诺还款日 | 最近有效承诺还款日，来源于承诺还款记录 |
| 最新跟进结果 | 最近一次跟进结果标签 |
| 最新联系时间 | 最近一次跟进时间 |
| 催收员 | 当前分配的催收员，未分配时标红 |

筛选：订单号 / 手机号、逾期天数区间（1-7 / 8-30 / 30+）、催收员、最新跟进结果。

操作：
- **记录跟进**：弹窗填写联系方式、联系结果、备注、下次跟进时间；结果为「承诺还款」时必填承诺日期和承诺金额，自动生成承诺还款记录。
- **分配催收员**：选择催收员、填写备注，更新任务分配关系并写入审计日志。

#### Tab 2：跟进记录

每次联系行为的完整流水。

| 字段 | 说明 |
|---|---|
| 跟进编号 | 系统生成唯一 ID |
| 借款订单号 | — |
| 客户 | 姓名 + 脱敏手机号 |
| 联系时间 | 实际联系时刻 |
| 联系方式 | 电话 / 短信 / WhatsApp / 系统推送 / 上门拜访 |
| 联系结果 | 已接听 / 未接听 / 承诺还款 / 拒绝还款 / 已还款 / 空号 |
| 承诺还款日 | 仅「承诺还款」结果时填写 |
| 备注 | 自由文本 |
| 催收员 | 操作人 |

筛选：订单号 / 手机号、联系结果、催收员。

#### Tab 3：承诺还款管理

管理所有 Promise-to-Pay 记录的履约状态。

| 字段 | 说明 |
|---|---|
| 承诺单号 | 系统生成唯一 ID |
| 借款订单号 | — |
| 客户 | — |
| 承诺金额 | 客户承诺的还款金额（PKR） |
| 承诺日期 | 客户承诺的还款日期 |
| 履约状态 | Pending / Fulfilled / Broken / Expired |
| 创建时间 | — |
| 催收员 | 跟进人 |

筛选：订单号 / 手机号、履约状态、承诺日期。

操作：
- **标记履约**：将状态改为 Fulfilled，写审计日志。
- **标记违诺**：将状态改为 Broken，写审计日志。
- 超过承诺日期仍为 Pending 的记录，前端显示「⚠ Overdue」提示。

**权限编码**

```
collection                           催收管理
  collection:tasks                   催收任务
    collection:tasks:view            查看催收任务列表
    collection:tasks:assign          分配催收员
    collection:tasks:followup        记录跟进
  collection:followups               跟进记录
    collection:followups:view        查看跟进记录
  collection:promises                承诺还款
    collection:promises:view         查看承诺还款
    collection:promises:update       标记履约/违诺
```

### 5.21 App 资源位管理

**角色权限：** OPERATIONS、SUPER_ADMIN

管理 App 各端资源位的展示内容和跳转配置，运营人员可实时更新 Banner、弹窗、入口图标等，无需发版。

#### Tab 1：资源位配置

| 字段 | 说明 |
|---|---|
| 位置名称 | 易读名称，如「首页顶部 Banner」 |
| 位置 Key | 唯一标识符，App 通过此 Key 拉取配置（创建后不可修改） |
| 类型 | Banner / Popup / Entry Icon / Float Button / Tab Item |
| 跳转类型 | H5（WebView）/ Native Page / External URL / No Action |
| 跳转链接 | URL 或 Native 路由 |
| 图片 URL | 图片资源地址 |
| 显示文案 | 按钮标签、Banner 标题等可选文字 |
| 排序 | 同类型资源位的展示顺序 |
| 生效时间 | 开始展示时刻（可为空，表示不限） |
| 失效时间 | 停止展示时刻（可为空，表示不限） |
| 目标用户 | All / New Users / Returning / Overdue Users |
| 状态 | Active / Inactive |
| 更新时间 | 最近修改时间 |

筛选：位置 Key / 名称关键字、类型、状态。

操作：
- **新增**：填写上述字段，Key 需唯一，创建后写入变更日志。
- **编辑**：可修改除 Key 以外的所有字段，保存后写入变更日志。
- **启用 / 禁用**：快速切换状态，写入变更日志。

#### Tab 2：变更记录

| 字段 | 说明 |
|---|---|
| 日志 ID | 系统生成 |
| 资源位 | 名称 + Key |
| 动作 | Created / Updated / Enabled / Disabled |
| 变更时间 | — |
| 操作人 | 管理员账号 |
| 变更前 | 变更前的关键字段快照 |
| 变更后 | 变更后的关键字段快照 |
| 备注 | 操作人填写的变更说明 |

筛选：位置 Key / 操作人、变更日期。

**权限编码**

```
config:app_resources                 App资源位管理
  config:app_resources:view          查看资源位列表
  config:app_resources:create        新增资源位
  config:app_resources:edit          编辑资源位
  config:app_resources:toggle        启用/禁用
  config:app_resources:log           查看变更记录
```

### 5.22 系统管理

**管理员账号**

- 字段：用户名、真实姓名、邮箱、手机号、角色、状态、最近登录、创建时间。
- 操作：创建、编辑、重置密码、启用/禁用、删除。
- 新密码仅展示一次。

**角色管理**

- 字段：角色编码、角色名称、描述、用户数、状态、排序、创建时间。
- 操作：创建、编辑、分配权限、查看用户、启用/禁用、删除。
- 已分配用户的角色禁止删除。

**权限管理**

- 树形展示菜单和按钮权限。
- 权限编码创建后不可修改。
- 已被角色使用的权限禁止删除。

**审计日志**

- 记录后台所有新增、编辑、删除、状态变更、审批、导出、查看敏感信息等操作。
- 字段：操作人、模块、动作、目标类型、目标 ID、IP、请求参数、结果、前后快照、时间。

## 6. 安全与合规

- 登录失败连续 5 次锁定 30 分钟。
- JWT 登录态，空闲超时时间 2 小时。
- 密码使用 bcrypt 或同等级安全算法存储。
- 手机号、证件号、联系人手机号、银行卡/钱包账号默认脱敏。
- 查看敏感信息、导出数据必须写入审计日志。
- 不同角色只能访问授权菜单和按钮。
- 删除均采用软删除。

## 7. 非功能需求

- 后台默认英文 UI，可支持中文切换。
- 业务时区：PKT（UTC+5）。
- 币种：PKR。
- 默认分页：20 条/页。
- 列表需支持筛选、分页，核心财务列表需支持导出。
- 所有金额字段保留 2 位小数。
- 枚举值由后端统一返回，前端仅做展示映射。
